# pdmt-web: Архитектурный аудит

## P0 — Баги, требующие немедленного исправления

- [v] **Race condition в refresh token flow** — два параллельных 401 вызывают два `tryRefresh()`, второй инвалидирует токен первого, пользователя выбрасывает из сессии. Решение: mutex/promise-коалесценция в `src/api/client.ts`
- [v] **Отсутствие AbortController в useEffect** — смена фильтров/недели не отменяет предыдущий запрос, старый ответ может перезаписать новый. Затронуты: `AnalyticsPage`, `CalendarPage`, `useEventList`, все 10 карточек `InsightsPage`

## P1 — Архитектурные проблемы

- [v] **Глобальный мутабельный стейт API-клиента** — `getToken`/`setToken`/`onLogout` в `src/api/client.ts` — module-level singletons, перезаписываются при каждом маунте AuthProvider. Нет гарантии порядка, невозможно тестировать изолированно. Решение: передавать token-getter через параметр или DI через Context
- [v] **Нет кеша/дедупликации запросов** — каждый переход на страницу перезагружает данные, два компонента с `getTags()` делают два запроса. Решение: TanStack Query (кеш, дедупликация, refetch, retry, AbortController из коробки)
- [v] **navigate() в render phase** — `LoginPage.tsx:17-19` вызывает `navigate()` в теле рендера (side-effect). В StrictMode — двойная навигация. Решение: перенести в `useEffect`

## P2 — Надёжность и поддерживаемость

- [v] **Дублирование паттерна загрузки в InsightsPage** — 10 карточек с идентичным шаблоном (useState x4 + useEffect x2), ~400 строк копипасты. Баг нужно чинить в 10 местах. Решение: хук `useLazyFetch<T>()` или TanStack Query
- [v] **Нет runtime-валидации ответов API** — `as Promise<T>` — type assertion, не проверка. Неожиданная структура ответа проявится как `Cannot read property of undefined` в рендере. Решение: Zod-схемы для критичных ответов
- [v] **Потеря контекста ошибок** — повсюду `catch { setError("Не удалось загрузить") }`, оригинальная ошибка (403/500/network) отбрасывается. Решение: маппинг статусов на пользовательские сообщения + логирование оригинала
- [v] **Content-Type: application/json на всех запросах** — `client.ts:46` ставит заголовок даже для GET/DELETE, может вызывать лишние CORS preflight. Решение: ставить только при наличии body

## P3 — UX и code quality

- [v] **Монолитный InsightsPage (837 строк → 70 строк)** — 10 независимых компонентов со своим стейтом и fetch в одном файле. Решение: вынести карточки в `src/pages/insights/`
- [ ] **Нет loading skeleton / Suspense** — текст "Загрузка..." вместо skeleton-ов, layout shift при каждом переходе
- [ ] **Нет retry-логики** — сетевая ошибка показывает сообщение без кнопки "Повторить", единственный выход — перезагрузка страницы
- [v] **eslint-disable в AnalyticsPage:323** — подавление `react-hooks/exhaustive-deps`, deps неполные (monday/sunday не указаны)
- [ ] **Дата-логика без библиотеки** — ручные вычисления дат хрупки при DST-переходах и таймзонах
